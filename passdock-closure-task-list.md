# PassDock 闭环任务清单

文档分类：

- 类型：开发归档
- 适用场景：回看项目开发收口过程、确认哪些闭环项已经完成
- 是否建议第一次优先阅读：否

- 如果你现在的目标是“把系统用起来”，不需要优先读这份

更新时间：2026-04-14

## 总体说明

这份清单用于确认项目是否已经从“页面原型 / 数据样例 / 半通路”推进到“真实可跑、可审核、可追踪、可运维”的完整闭环。

当前状态：

- 1 至 13 项均已完成
- 主业务链路已经闭环
- 当前后续工作重点更偏向部署、运维和上线环境接入，而不是核心业务缺口

## 1. 已完成：人工交付闭环

- 目标：人工交付和邮件交付订单在支付完成后不再被错误地自动标记为完成
- 范围：
  - 人工 / 邮件交付保持在 `delivery_pending`
  - 写入排队中的交付记录
  - 后台支持人工完成交付
  - 补齐后端测试

## 2. 已完成：OKX USDT 内建扫单闭环

- 目标：`okx_usdt` 不再只依赖回调消费，而是具备内建 watcher / job 驱动能力
- 范围：
  - 增加 watcher 调度与轮询入口
  - 复用 `/internal/v1/payments/onchain/confirm`
  - 支持通过环境变量和运行时设置进行控制

## 3. 已完成：真实 new-api 发码闭环

- 目标：生产环境不再静默回退到 mock 目标
- 范围：
  - 为 `mock://` Provider 增加启动和运行时保护
  - 开发环境保留引导便利
  - 非开发环境配置错误时快速失败
  - 对 Provider 健康状态暴露更清晰的信号

## 4. 已完成：重试策略与异步队列执行闭环

- 目标：让配置中的重试策略和异步设置真正进入业务执行路径
- 范围：
  - 履约和交付使用真实重试策略
  - 增加 DB 驱动的延迟 / 重试调度
  - 异步配置不再只是占位

## 5. 已完成：支付凭证对象访问控制闭环

- 目标：避免支付凭证原始对象 URL 被直接公开访问
- 范围：
  - 前台需要订单级访问令牌
  - 后台需要管理员鉴权
  - 保留当前上传与预览体验
  - 补齐访问控制测试

## 6. 已完成：Telegram Webhook 运维闭环

- 目标：让后台系统页能真正管理 Telegram Webhook
- 范围：
  - 后台查看配置中的 Webhook 设置
  - 后台查看 Telegram 实时 Webhook 信息
  - 支持同步、删除、删除并清空待处理更新
  - 本地运行时文案与真实行为对齐

## 7. 已完成：后台支付凭证预览闭环

- 目标：运营人员在后台看到真实上传文件，而不是只有元数据
- 范围：
  - 前端支持管理员鉴权下载二进制文件
  - 支付审核表格与详情抽屉支持预览入口
  - 支持图片 / PDF 预览
  - 本地草稿预览依然可用

## 8. 已完成：前台支付凭证预览闭环

- 目标：买家在站内可以查看自己提交的支付凭证
- 范围：
  - 统一的前台支付凭证预览弹层
  - 结算页、订单中心、账号中心复用
  - 继续使用订单级访问令牌保护文件

## 9. 已完成：生产引导数据与演示业务数据拆分

- 目标：生产 / 预发布环境首次启动不再写入演示订单与示例凭证
- 范围：
  - 参考数据与演示业务数据拆分
  - 启动逻辑按环境默认行为执行
  - 开发 / 测试环境继续保持方便
  - 补齐回归测试

## 10. 已完成：真实 new-api 上游部署接入闭环

- 目标：让 PassDock 的 Provider / Action 模板能直接对齐真实上游部署
- 范围：
  - 保持内部适配契约稳定
  - 对外暴露真实配置项和部署说明
  - 无需改代码即可切换签名上游

## 11. 已完成：微信 / 支付宝回调适配闭环

- 目标：二维码人工审核通道不再只能吃 PassDock 自己的统一回调格式
- 范围：
  - 新增按通道 / Provider 的回调字段映射
  - 保留人工二维码兜底流程
  - Telegram 和后台审核流程不受影响

## 12. 已完成：集成中心安全 live test 闭环

- 目标：让后台可以执行安全受控的真实联调，而不是只能看预览
- 范围：
  - 区分 `preview` / `auto` / `live`
  - 保护生产型外部写操作
  - 返回更清晰的执行元信息

## 13. 已完成：运维可观测性与文档闭环

- 目标：让项目上线后具备基本的探活、可观测性和部署文档基础
- 范围：
  - API 草案与配置说明补齐
  - 增加 `healthz` / `readyz` / `metrics`
  - 增加统一错误上报接口
  - 记录上传、Webhook、上游 Provider 的部署说明
